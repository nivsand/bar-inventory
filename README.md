# Bar Inventory, Ordering & Prep Management System

Mobile-first web app for managing inventory, supplier orders, deliveries, prep
production and daily counts in a bar/restaurant. Simple for employees, powerful
for managers. Hebrew (RTL) + English (LTR).

## Stack
Next.js 14 (App Router) · React 18 · TypeScript · Tailwind CSS · Prisma · PostgreSQL · NextAuth (JWT credentials).

## Quick start

```bash
# 1. Install
npm install

# 2. Configure environment
cp .env.example .env
#   set DATABASE_URL + DIRECT_URL (same value on plain Postgres),
#   NEXTAUTH_SECRET (openssl rand -base64 32), and NEXTAUTH_URL to your dev port.

# 3. Create the schema + seed realistic demo data
npx prisma migrate dev --name init      # creates a versioned migration
npm run db:seed

# 4. Run
npm run dev      # http://localhost:3000
```

> If you run on a non-default port (e.g. `next dev -p 3001`), set
> `NEXTAUTH_URL=http://localhost:3001` so auth redirects resolve correctly.

## Importing your existing Excel data
Nothing is hard-coded — the seed is only starter data. To load your spreadsheets:

```bash
npm run import:excel -- ./path/to/your-file.xlsx
# dry run (prints parsed rows, no DB writes):
DRY_RUN=1 npm run import:excel -- ./path/to/your-file.xlsx
```

The importer (`scripts/import-excel.ts`) matches columns case-insensitively in
Hebrew or English. Edit `COLUMN_ALIASES` there to map your exact headers.
Expected sheets: a *Suppliers* sheet and an *Inventory/Items* sheet (see the
file header for column names). Everything imported is editable in the UI.

## OCR
The MVP ships a deterministic stub provider so the receipt-review flow works with
zero API keys. To wire a real provider, implement `OcrProvider` in
`src/server/ocr/index.ts` and set `OCR_PROVIDER` + `OCR_API_KEY`. Inventory is
**never** updated automatically — extracted items go to a manager review screen.

## Install as a phone app (PWA)
The app is a Progressive Web App: a web manifest, service worker, theme color and
maskable icons are included.
- **iPhone (Safari):** Share → *Add to Home Screen*.
- **Android (Chrome):** menu → *Install app* / *Add to Home screen*.

It then launches full-screen with the martini icon, respects iPhone safe areas,
and the daily-count and waste screens use large, phone-friendly inputs (16px+ so
iOS never zooms on focus). The service worker only registers in production
builds. Icons are generated by `node scripts/gen-icons.mjs` (no image libraries
required) into `public/icons/`.

## Deployment (Vercel + Neon)

### Environment variables (set all in Vercel → Project → Settings → Environment Variables)
| Variable | Required | Notes |
|----------|----------|-------|
| `DATABASE_URL` | ✅ | Neon **pooled** connection string (host contains `-pooler`). Used at runtime. |
| `DIRECT_URL` | ✅ | Neon **direct** connection string. Used by `prisma migrate deploy`. |
| `NEXTAUTH_SECRET` | ✅ | `openssl rand -base64 32`. |
| `NEXTAUTH_URL` | ✅ | Your production URL, e.g. `https://your-app.vercel.app`. |
| `OCR_PROVIDER` | ⬜ | Leave blank for the built-in stub. |
| `OCR_API_KEY` | ⬜ | Only if using a real OCR provider. |

### Steps
1. **Create the Neon database** and copy both connection strings (pooled →
   `DATABASE_URL`, direct → `DIRECT_URL`). Keep `?sslmode=require`.
2. **Push the repo to GitHub** and import it into Vercel (framework auto-detected
   as Next.js). Add the env vars above.
3. **Apply migrations to production** (production-safe — applies committed
   migrations, never drops data like `db push` can):
   ```bash
   # from your machine, with prod DIRECT_URL exported, OR via Vercel build:
   npm run prisma:deploy        # = prisma migrate deploy
   ```
   You can also add `prisma migrate deploy` to the Vercel **Build Command**
   (`prisma generate && prisma migrate deploy && next build`) so each deploy
   applies pending migrations automatically. `prisma generate` already runs on
   `postinstall`.
4. **Seed once (optional)** for the initial admin/demo data:
   ```bash
   DATABASE_URL=... DIRECT_URL=... npm run db:seed
   ```
   Then change the demo passwords from the Users screen.
5. **Deploy.** Vercel builds with `npm run build`. NextAuth uses `NEXTAUTH_URL`
   for canonical links; logout uses a relative redirect so it stays correct on
   preview URLs and custom domains.

### Why this is production-safe
- Prisma client is generated on `postinstall` and in `build` (Vercel caches
  `node_modules`, so a generate step is required).
- Migrations use `prisma migrate deploy` against the **direct** (non-pooled)
  Neon URL — pooled connections can't run DDL reliably.
- The app connects through the **pooled** URL, sized for serverless.
- Count submission runs in a single bounded transaction with bulk reads, so it
  won't hit interactive-transaction timeouts under Neon latency.
